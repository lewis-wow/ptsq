# Server side calls

Server side calls are for calling the `route` (mutation or query) directly on the server.
They also run all the middlewares registered to the resolver that creates the `mutation` or `query` unlike calling `resolve` function of the resolver only.

```ts title="server.ts"
const { middleware, resolver, router, serve } = createServer({
  ctx: createContext,
});

const testMutation = resolver.args({ name: z.string() }).mutation({
  output: z.string(),
  resolve: async ({ input /* { name: string } */ }) /* string */ => {
    return `Hello, ${input.name}`;
  },
});

const baseRouter = router({
  test: testMutation,
});

const caller = baseRouter.createCaller({
  /* server context */
});

const result = await caller.test.mutate({
  name: /* string */ 'John',
});
```

The `Router.createCaller` function expects context object as the parameter, because there is no context on the server side without request from the client.
